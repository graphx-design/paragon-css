Paragon CSS Framework

Introduction

There are already plenty of CSS frameworks, with varying levels of complexity. This one takes the Occam's Razor approach: it is minuscule, restricted to pure CSS. It strays from the extreme simplicity of the flat layered "paper" trend, in favor of more classic visual cues enhanced to use modern browser capabilities.

Almost all classes begin with the prefix pgn- to prevent collisions with other frameworks. For brevity, there is no prefix for attribute classes like is-something, has-something and size-something.

Compiled CSS files are located in dist/. You may use all.css to get the entire framework at once, or just some of the individual files described below. Be sure to import core.css first in order to avoid priority side-effects.

Browser Support

In addition to all currently supported browsers, including "long term support" versions, in order to support our enterprise environments we're supporting Microsoft Internet Explorer 11. This does not incur any significant cost, except that our design avoids the use of CSS variables, calc(), mix-blend-mode and filter(). We intend to refactor to make use of these features when we'll drop IE11 support.

Responsive

Certain classes restrict which devices can see an element, by viewport width or display type. Note that the names "mobile", "tablet" and "desktop" are used as nicknames for the three responsive size ranges, not as true device identification. CSS media queries also use a "device pixel ratio" to define their own meaning for "px", so those are not actual pixels.

Mobile:
Up to 767px, .pgn-mobile-only, .pgn-tablet-down
Tablet:
From 768px to 991px, .pgn-tablet-only, .pgn-tablet-up, .pgn-tablet-down
Desktop:
From 992px up, .pgn-tablet-up, .pgn-desktop-only
Print:
.pgn-print-only, .pgn-screen-only (non-screen is considered "print")

Block .pgn-block

Makes any element (including buttons) a block and full width.

Generic Container .pgn-container

The .pgn-container class pads a block and limits its width on large viewports.

Layouts .pgn-l-*

Applicable to any container block, including body.

.pgn-l-3
3 children, stacked vertically. The middle child expands as necessary to make sure that the third child is always no higher than the bottom of the viewport. Typical header-main-footer layout.

Grid grid.css

You may import grid-10.css, grid-24.css or both in addition to the base grid.css in your project. Those two bases were chosen for their versatility: areas can be divided by 2, 3, 4, 5, 6, 8, 10, 12 and 24.

Rows are .pgn-row-10 or .pgn-row-24 according to whether you want the row to be divided in 10ths or 24ths. Columns are .pgn-col-X where X is the multiple width. Rows wrap, so you only need to use multiple row blocks for splitting incomplete lines.

.pgn-col-* padding is sensitive to .pgn-density-* higher in the DOM tree

An <img> directly inside a .pgn-col-* automatically gets full width.

Responsive

.pgn-doubling .pgn-col-*
Makes grid cells double in relative width when stepping down from "desktop" to "tablet" size, and double again down to "mobile" size. (i.e. A 1/12th cell would become 1/6th wide in "tablet" viewports and 1/3rd wide in "mobile" viewports.)
.pgn-stacking .pgn-col-*
Makes grid cells go full-width for "tablet" and "mobile" sizes.

Forms forms.css

Note that buttons and input widgets with :focus get a z-index of 1000 or 1001 in addition to the halo.

Fields label.pgn-field .pgn-label + input|textarea

In addition to text areas, typical input types compatible with this class are: email, number, password, tel, text, url and possibly search. Fields expand to fill their label's width, so a label may be or be inside a grid cell. We implement infield top aligned labels for their clarity.

Field sets

Checkboxes label input[type=checkbox] + .pgn-label

Radio selections label input[type=radio] + .pgn-label

General rules of thumb: if there are between 3 and 5-7 options, a radio is probably the best input. Stack vertically if possible; space generously if horizontal. For 2 options, a checkbox is clearer. For more than 5-7 options, a select drop-down keeps the form cleaner. Remember to always check a default option when presenting a form's initial state.

Drop-down menus TODO

General rules of thumb: if there are no more than 5-7 options and the form has enough room, consider using radio buttons, which have a lower cognitive load.

Buttons .pgn-form button, .pgn-button

Icon Buttons .pgn-icon-button

Unlike other form styles, icon buttons have fixed sizes based on touch display ability and their priority. Normal by default, they may have .is-secondary or .is-priority. On touch devices, these three sizes are 11mm, 15mm and 19mm.

Button groups .pgn-group button, .pgn-group .pgn-button

Theming

Normal buttons automatically adapt to any background/foreground colors and font sizes set on their elements, as well as their parent background color. Icon buttons adapt to font size and their parent background color.

M

Responsive

For touch devices, all buttons have a minimum width and height of 11mm regardless of font size.

Transitions transitions.css

To guarantee that no browsers "flash" transitions on load, some rules depend on a parent having the "loaded" class. To use them, add the loaded class to body after the first paint is likely over. In some browsers, a simple "onload" is insufficient, thus:

<body onload="setTimeout(function(){document.body.classList.add('loaded')})">

Generic

.loaded, .loaded *
Subtle fade for color, background-color and opacity.
.pgn-go-hidden, .pgn-go-visible
Fade in/out of opacity. Only useful in scripts because they are mutually exclusive.
